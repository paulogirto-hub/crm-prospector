# FRONT-46 - Design Emocional e Psicologia do Usuário

> **Prioridade:** MEDIO
> **Depende de:** FRONT-30
> **É dependência de:** Todas as interfaces (Frontend/Mobile).
> **Categoria:** frontend

## 1. Psicologia na Interface

Sistemas profissionais devem influenciar o comportamento do usuário de forma positiva e ética.

### Gatilhos e Nudges (Empurrões)
- **Social Proof (Prova Social):** Exibir métricas de sucesso ou confiança em momentos críticos (ex: "X usuários usaram este fluxo com sucesso").
- **Dopamine Loops (Micro-recompensas):** Animações de sucesso, sons sutis e mensagens de "Parabéns" ao concluir tarefas complexas.
- **Endowment Effect:** Permitir personalização rápida para que o usuário sinta que o sistema "é dele" desde os primeiros segundos.

---

## 2. Emotional UI (Design de Sentimentos)

O design deve evocar estados emocionais específicos baseados no nicho:
- **Sistemas de Cyber/Bancários:** Devem evocar *Segurança e Sobriedade* (Azuis profundos, tipografia robusta, feedbacks claros de "Protegido").
- **Sistemas de Criatividade/IA:** Devem evocar *Exploração e Magia* (Gradientes, transparências, micro-animações fluidas).

---

## 3. Redução de Carga Cognitiva

- **Lei de Hick:** Quanto mais opções, mais tempo o usuário leva para decidir. Simplificar menus e fluxos.
- **Progressive Disclosure:** Mostrar apenas o necessário no momento. Esconder funções avançadas atrás de botões de "Configurações" ou "Avançado".

---

## 4. Design para a Diversidade (Acessibilidade Ética)

Não é apenas sobre cores, é sobre inclusão cognitiva:
- **Leitura Fácil:** Uso de linguagem clara, sem jargões desnecessários.
- **Suporte a Daltonismo:** Nunca usar apenas a cor para transmitir informação (ex: usar ícone + cor para erros).
- **Consistência Motora:** Botões em locais previsíveis para facilitar a memória muscular.

---

## 5. Checklist de Design Emocional

- [ ] O sistema passa sensação de "Velocidade" através de feedbacks instantâneos?
- [ ] Há um tom de voz consistente em todas as mensagens (UX Writing)?
- [ ] O usuário sabe exatamente onde clicar para a próxima ação (Call to Action)?
- [ ] O sistema "perdoa" erros através de botões de Undo/Desfazer claros?
